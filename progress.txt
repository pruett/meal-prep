### Files added

- convex/meals.ts (3 exported functions: create, getByMealPlan, updateStatus)

### Files changed
- convex/schema.ts (expanded from 1 table to 6 tables with full field definitions and indexes)
- PLAN.md (marked 3 tasks complete)

### Notes
- `meals.getByMealPlan` collects all meals then sorts in-memory by `sortOrder` since Convex index ordering is by the indexed field, not an arbitrary sort field
- `mealPlans.getByUserAndWeek` uses `.unique()` since there should only be one plan per user per week
- `mealPlans.getByUser` orders descending so most recent plans appear first
- Schema uses `v.union(v.literal(...))` pattern for enums (status fields, provider, type)
- `meals.fullRecipe` is `v.union(v.object(...), v.null())` — null until prep guide generation populates it
